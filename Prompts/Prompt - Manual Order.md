# درخواست طراحی صفحه جدید

## اطلاعات پایه
- نام صفحه: ایجاد سفارش دستی
- هدف: فروشنده می‌تواند به صورت مرحله‌ای یک سفارش 
  جدید برای مشتری ایجاد کند و لینک پرداخت بگیرد
- Breakpoint هدف: 1440
- نوع صفحه: Wizard / Multi-step form

## ساختار Wizard — 5 مرحله

### Step Indicator (بالای همه مراحل — ثابت)
- نمایش ۵ مرحله به صورت افقی
- مرحله فعال: رنگ teal/solid
- مراحل تکمیل‌شده: علامت تیک + teal/subtle
- مراحل باقی‌مانده: gray/muted
- Component: Steps از Chakra UI

### مرحله ۱ — انتخاب مشتری
محتوا:
- Search input برای جستجوی مشتری موجود
  (جستجو بر اساس نام یا شماره موبایل)
- لیست نتایج جستجو با Card برای هر مشتری
  (نام + شماره موبایل + آدرس)
- Divider با متن «یا»
- Button برای «ایجاد مشتری جدید»
  که یک mini-form نمایش می‌دهد:
  نام، نام خانوادگی، شماره موبایل (required)

State ها:
- حالت اولیه: فقط search box + دکمه ایجاد
- حالت جستجو: نمایش skeleton loading
- حالت نتایج: لیست مشتریان
- حالت انتخاب‌شده: Card مشتری با border teal

### مرحله ۲ — انتخاب محصول
محتوا:
- Filter bar: دسته‌بندی (Select) + جستجو (Input) + 
  وضعیت موجودی (SegmentedControl: همه / موجود)
- Grid محصولات: 2 ستون
  هر Card: عکس + نام + موجودی + قیمت + انتخاب تنوع (اگر داشت)
- Counter برای تعداد هر محصول (InputNumber)
- سینی انتخاب‌شده‌ها در پایین (sticky): 
  تعداد آیتم‌ها + جمع موقت قیمت

State ها:
- محصول ناموجود: disabled + badge «ناموجود»
- محصول انتخاب‌شده: border teal + counter فعال
- موجودی کم: badge warning «فقط X عدد»

### مرحله ۳ — روش ارسال
محتوا:
- لیست روش‌های ارسال (Radio Card برای هر روش)
  هر کارت: عنوان + نوع (درون/برون‌شهری) + 
  نوع کرایه (پیش‌کرایه/پس‌کرایه) + قیمت ارسال
- بخش آدرس گیرنده:
  استان (Select) + شهر (Select) + 
  کدپستی (Input) + آدرس کامل (Textarea)
- یادداشت داخلی (Textarea — اختیاری)
  توضیح: «این یادداشت فقط برای شما قابل مشاهده است»

### مرحله ۴ — تخفیف (اختیاری)
محتوا:
- عنوان «اعمال تخفیف (اختیاری)»
- لیست کدهای تخفیف فعال به صورت Card:
  نام تخفیف + مقدار (درصد یا ریال) + 
  تاریخ انقضا + محدودیت استفاده
- Badge برای کدهایی که به حد محدودیت رسیدن: disabled
- دکمه Skip: «بدون تخفیف ادامه بده»

### مرحله ۵ — تأیید و ایجاد لینک
محتوا:
- خلاصه سفارش (read-only):
  - اطلاعات مشتری
  - لیست محصولات با تعداد و قیمت
  - روش ارسال و هزینه
  - تخفیف (اگر انتخاب شده)
  - جمع کل پرداختی
- دکمه اصلی: «ایجاد سفارش و دریافت لینک پرداخت» (teal)
- بعد از ایجاد: نمایش Modal با:
  - لینک پرداخت
  - دکمه کپی لینک
  - دکمه ارسال پیامک (اختیاری)

## قوانین خاص این صفحه
- Wizard باید همیشه Step Indicator نشان بدهد
- کاربر باید بتواند به مراحل قبلی برگردد
- هر مرحله Footer ثابت دارد: دکمه «مرحله قبل» + «مرحله بعد»
- مرحله ۲: قبل از رفتن به مرحله بعد موجودی re-check بشه
- مرحله ۴ اختیاریه — دکمه Skip باید واضح باشد
- در مرحله ۵ هیچ فیلدی قابل ویرایش نیست
