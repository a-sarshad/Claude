# Health App — Project Brief & Claude Instructions

## Project Description

یک اپلیکیشن هوشمند در حوزه بهداشت و سلامت که به‌عنوان یک **دستیار شخصی AI** عمل می‌کند. هدف اصلی این اپ کمک به کاربران برای بهبود کیفیت زندگی از طریق راهنمایی‌های شخصی‌سازی‌شده در حوزه‌های تغذیه، فعالیت بدنی، سلامت روان، خواب و سایر جنبه‌های سلامت است.

این اپ به‌صورت **PWA (Progressive Web App)** طراحی می‌شود تا روی تمام دستگاه‌ها (موبایل، تبلت، دسکتاپ) قابل استفاده باشد و در آینده قابلیت تبدیل به Native App را داشته باشد.

---

## Tech Stack

- **Frontend**: React + TypeScript
- **UI Library**: Chakra UI v3 (فارسی، RTL)
- **Font**: Vazirmatn
- **AI Engine**: Anthropic Claude API (claude-sonnet-4-20250514)
- **Platform**: PWA (اول)، Native App (در آینده)
- **Design System**: Chakra UI FA (رجوع به ds-chakra-ui-fa skill)

---

## Design Direction

- **سبک**: Modern Medical + Warm Tech — ترکیبی از دقت پزشکی و گرمای انسانی
- **رنگ اصلی**: Teal (`#0D9488`) — نماد سلامت، اعتماد، تازگی
- **تایپوگرافی**: Vazirmatn — برای فارسی، خوانا و مدرن
- **جهت**: RTL (راست به چپ) — برای مخاطب فارسی‌زبان
- **مود**: Light Mode به‌عنوان پیش‌فرض، Dark Mode در آینده
- **تجربه کاربری**: باید برای تمام سنین جذاب باشد — هم جوان‌ها، هم میانسالان

## Claude Instructions for This Project

### زبان و لحن
- همیشه به **فارسی** پاسخ بده مگر اینکه کاربر به زبان دیگری بپرسد
- لحن باید **دوستانه، حرفه‌ای و انگیزه‌بخش** باشد

### طراحی صفحات (Artifact/HTML)
- چون Chakra UI در artifact قابل استفاده نیست، از **HTML + Tailwind CSS** یا **Vanilla CSS** استفاده کن
- رنگ‌ها و فونت‌ها را مطابق Design System پروژه انتخاب کن:
  - Primary: `#0D9488` (teal)
  - Background: `#FFFFFF` / `#F8FAFC`
  - Text: `#0F172A` / `#52525B`
  - Font: Vazirmatn از Google Fonts
- همیشه `dir="rtl"` روی root element
- طراحی باید **Mobile-First** باشد (PWA)

### وقتی صفحه طراحی می‌کنی
1. ابتدا **ساختار کلی صفحه** را با توضیح بنویس
2. سپس **کد HTML/CSS** را در artifact جداگانه بساز
3. نام artifact را همیشه متفاوت انتخاب کن (مثلاً `health-app-home-v1`, `health-app-profile-v2`)
4. طراحی باید **واقعی و قابل استفاده** باشد، نه mock ساده

### وقتی prompt برای Figma Make / claude.ai/design می‌نویسی
- دقیق و جامع باشد
- رنگ‌ها را به‌صورت hex بنویس
- component‌های Chakra UI را نام ببر
- layout و spacing را مشخص کن

### کد Chakra UI (برای توسعه واقعی)
- همیشه از Chakra UI v3 استفاده کن
- RTL را حفظ کن
- از font Vazirmatn استفاده کن
- Token‌های design system را رعایت کن (رجوع به ds-chakra-ui-fa skill)

### محدودیت‌های مهم
- **هرگز** توصیه پزشکی قطعی نده — همیشه توصیه به مشورت با پزشک را در پیام‌های AI داشته باش
- اطلاعات کاربر باید **محرمانه** تلقی شود
- AI باید **empathetic** و **supportive** باشد، نه صرفاً اطلاعاتی

## نکات فنی PWA

- باید `manifest.json` داشته باشد
- Service Worker برای offline mode
- Responsive برای همه اندازه‌های صفحه
- Dark Mode آماده برای آینده (CSS variables)
- Performance: Lazy loading برای صفحات
